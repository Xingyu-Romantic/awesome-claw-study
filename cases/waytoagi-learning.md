# 实战案例：WaytoAGI 知识库学习

> 使用 OpenClaw Agent 批量学习知识库内容

## 场景

用户想要学习 WaytoAGI 知识库（https://waytoagi.com/），但内容太多，希望用 Agent 自动抓取和学习。

## 实现方案

启动 5 个并行的 Sub-Agent，每个负责一个板块：

| Agent | 板块 | 任务 |
|-------|------|------|
| waytoagi-agent-1 | AI入门系列 | 递归学习子主题 |
| waytoagi-agent-2 | AI精选系列 | 收集网站/视频/播客 |
| waytoagi-agent-3 | AI应用与前沿 | 跟踪最新应用 |
| waytoagi-agent-4 | AI Agents 智能体 | Agent 技术深度学习 |
| waytoagi-agent-5 | AI创意与创作 | 创意案例收集 |

## 技术要点

### 1. Sub-Agent 启动

```bash
sessions_spawn --task "学习 WaytoAGI 的 AI入门系列板块，提取关键知识点" \
  --label waytoagi-agent-1 \
  --runtime subagent
```

### 2. 递归学习

每个 Agent 需要：
- 访问主页面获取所有子链接
- 逐个访问子链接提取内容
- 递归深入（子链接的子链接）

### 3. 结果汇总

学习完成后自动创建飞书文档：

| 主题 | 文档链接 |
|------|----------|
| AI入门系列：AI学习路径 | 飞书文档链接 |
| AI入门系列：a16z AI典藏文章 | 飞书文档链接 |
| AI精选系列：AI网站精选 | 飞书文档链接 |
| ... | ... |

## 经验总结

1. **大任务需要时间** - 知识库内容庞大，需要让 Agent 有足够时间
2. **并行加速** - 多个 Agent 同时运行可以大幅缩短时间
3. **结果可能不完整** - 需要检查文档是否成功创建
4. **递归深度控制** - 避免陷入无限深层链接

## 改进方向

1. 添加进度报告 - 每隔一段时间汇报当前进度
2. 添加超时控制 - 防止单个任务运行过长
3. 添加断点续传 - 支持从中断处继续
4. 结果验证 - 自动检查文档是否成功创建
